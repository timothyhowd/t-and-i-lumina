# Lumina — local setup

Steps to get the PoC running on your machine. Run once.

## Prerequisites

- **Node.js 20+ and npm.** Install from [nodejs.org](https://nodejs.org/) or `brew install node`. The PoC has been verified against Node 26.x / npm 11.x.
- **A Portkey gateway API key** (DoorDash internal). Issued from the Portkey UI — the company's centrally-managed LLM gateway at `cybertron-service-gateway.doordash.team`. *Not* a personal Anthropic API key.

## One-time setup

```bash
cd ~/Development/lumina/app

# 1. Install JS dependencies
npm install

# 2. Configure environment
cp .env.local.example .env.local
# Edit .env.local — paste your Portkey API key into PORTKEY_API_KEY.
# Leave LUMINA_POC_MODE=true unless you have a reason to disable it.
```

Open `.env.local` in any editor:

```bash
open -e ~/Development/lumina/app/.env.local
```

Change `PORTKEY_API_KEY=pk-...` to your real key. Save.

## Run the app

```bash
cd ~/Development/lumina/app
npm run dev
# Open http://localhost:3000
```

## Health checks

```bash
# TypeScript types
npm run typecheck

# Production build
npm run build
```

## What you should see

1. The root page (`/`) has the red PoC banner across the top, a diagonal "PROTOTYPE — NOT FOR EXECUTION" watermark in the background, and a "Start a draft →" button.
2. The draft page (`/draft`) is the chat surface for Agent 3 (the conversational front-end that orchestrates Agents 1 and 2).
3. Without `LUMINA_POC_MODE=true` set, generation endpoints will refuse to run — by design, see [`docs/POC-LIMITATIONS.md`](./POC-LIMITATIONS.md).

## Troubleshooting

- **`npm install` fails** — verify Node is installed: `which node` should print a path (typically `/usr/local/bin/node`).
- **The chat UI says "POC mode is off"** — your `.env.local` is missing `LUMINA_POC_MODE=true`.
- **Generation endpoints return `PORTKEY_API_KEY is not set`** — set it in `.env.local`. Restart `npm run dev` after editing — Next.js reads env vars at startup, not per request.
- **Generation returns `model not found` or similar from the gateway** — the model slugs default to `@anthropic/claude-opus-4-7` and `@anthropic/claude-haiku-4-5`. If your Portkey Model Catalog uses different slugs, override in `.env.local`:
  ```
  LUMINA_MODEL_OPUS=@anthropic/claude-opus-4-7
  LUMINA_MODEL_HAIKU=@anthropic/claude-haiku-4-5
  ```
- **Connection refused / DNS errors to `cybertron-service-gateway.doordash.team`** — you need to be on the corporate network (VPN or in-office).
- **Drive sync** — `corpus/drive-inventory.json` is generated by walking the shared Drive (folder ID `0AGC8z3-x7H0kUk9PVA`). The committed inventory was generated on 2026-05-12.

## Notes on the Portkey path

The Anthropic-native SDK (`@anthropic-ai/sdk`) was the original choice, but DoorDash policy routes all LLM traffic through Portkey, which exposes an OpenAI-compatible `/v1/chat/completions` interface. As a result, the PoC drops these Anthropic-native features:

- **Prompt caching** (`cache_control: ephemeral`) — Portkey's OpenAI compat layer does not pass Anthropic's cache headers through. Each turn pays full input-token cost.
- **Adaptive thinking** (`thinking: adaptive`) — not expressible in OpenAI chat-completions schema.
- **Effort parameter** — same.

For PoC scope these are acceptable. If/when DoorDash's Portkey exposes an Anthropic-native passthrough endpoint, `app/lib/anthropic.ts` is the only file we'd swap back — everything downstream of it is provider-agnostic.
